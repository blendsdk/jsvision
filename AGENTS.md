# Project guidance

- This is a Node 22+, ESM TypeScript monorepo managed with Yarn 1 workspaces and Turborepo.
- Public SDK packages live under `packages/`; consumer-facing documentation lives in
  `packages/docs-site/`.
- Use `yarn verify` as the authoritative full gate. Use package-local `typecheck`, `test`, or
  `check:docs` commands for focused iteration.
- `tools/jsvision-skill/` is the canonical agent-neutral JSVision skill.
  `plugins/jsvision-plugin/skills/jsvision/` is generated by `yarn plugin:update`; never edit that
  distributed copy directly.
- Treat the Codex plugin as a supported SDK surface. After implementing any feature or bug fix under
  a path mapped by `tools/jsvision-plugin-impact.json`, review every reported skill reference, run
  `yarn plugin:update`, and include those updates in the same commit.
- Run `yarn plugin:check` before considering an SDK change complete. CI is a backstop, not the first
  place plugin drift should be discovered.
- `yarn plugin:update` owns generated API pages, synchronized recipe snippets, the source-impact
  snapshot, and the assembled Codex skill copy.
- Keep consumer plugin scripts independent of this monorepo at runtime. Resolve JSVision and
  development tools from the consumer project, validate paths, and never install dependencies
  without the workflow's explicit authorization.
- CodeOps lifecycle artifacts and configuration live under `codeops/`.
