### Resolution Notes

- **AR-1 / AR-2** were decided in the sweep-scoping round that preceded requirements authoring; they set the whole direction and are carried in verbatim.
- **AR-4** is the subtle one: in this framework keyboard focus traversal follows the view-tree child add-order, so a flex rebuild that re-orders `.add()` calls can silently change tabbing. The invariant forces the rebuild to preserve the logical order; each dialog gets a traversal-order spec test so a regression fails loudly.
- **AR-10** is the headline outcome: this is a net code *deletion*, not an `at()`-for-`at()` swap. The min-size hypothesis was disproved — grow-mode captured no per-child minimum, so nothing is lost by removing it.
- **AR-11** bounds the set: the divergence policy does not touch datagrid or the layout primitive; those remain ordinary epic children.
