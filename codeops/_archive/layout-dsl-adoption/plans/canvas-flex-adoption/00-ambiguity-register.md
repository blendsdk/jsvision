## Runtime additions (preflight, 2026-07-19)

| # | Item | Decision | Status |
|---|------|----------|--------|
| AR-13 (runtime) | **The witness seam.** The original strategy had every witness *reconstruct* the demo's view tree inside the test file. Both preflight auditors independently found that a reconstruction never executes a line of the converted file, so it passes after the conversion by construction — leaving 5 of 9 files with no oracle while the plan advertised the witnesses as its safety net. | **Per-file seam, each observing the real artifact** (user ruling): frame snapshots for the demos that already print their composed buffer and are already spawned by the e2e harness — a geometry oracle at zero source change; the already-exported `drillDownStory` for the story. **Refined by AR-16**: `editor-demo` also prints a frame (so it needs no source change), and `chrome-bars-demo` gets a sibling module rather than an export. Recorded in [07 §seam rule](07-testing-strategy.md). | ✅ Resolved |
| AR-14 (runtime) | **ST-C10 cannot be both green-first and a direction detector** if it mounts a panel standalone: neither builder sets `direction` today, so it solves as a row (engine default) and the witness is red-first — while a witness that supplies the direction itself keeps supplying it after task 2.4 and detects nothing. | **Drive `createDesignerApp({ caps, viewport, requireTty:false })`** and assert against the real app tree, which is green today and genuinely red if task 2.1/2.2 fails to move the direction into the builder. Split into ST-C10a/b/c so tasks 2.3 and 2.6 name distinct legs. | ✅ Resolved |
| AR-15 (runtime) | **The `background` folds have no oracle.** `Group.background` fills style only (`group.ts:76-77` — `ctx.fill(' ', …)`, the char stays `' '`), and every frame-snapshot seam prints `cell.char`. So 6 of 32 conversions would change a property no witness could see, while NFR-1 promises rendered output is unchanged. | **Narrow the folds to the 2 an importing witness can observe** (user ruling); the other 4 stay separate `.background =` assignments. Preserves NFR-1's guarantee at the cost of a cosmetic deletion the issues never asked for. | ✅ Resolved |
| AR-16 (runtime) | **The `buildRoot()` export seam is unimplementable.** Both candidate files invoke their entrypoint at module scope, and `chrome-bars-demo/main.ts:118-124` calls `process.exit()` there — importing it would kill the vitest worker. And the justification was false: `editor-demo` *does* print its composed frame (a local helper named `frame()`, 9 calls) and *does* have an e2e test. | **`editor-demo` moves to the frame-snapshot seam** (zero source change), and `chrome-bars-demo` gets a **sibling `tree.ts` module** that `main.ts` imports — no entry guard, no exit hazard, `main.ts` stays a pure entrypoint (user ruling). | ✅ Resolved |
| AR-17 (runtime) | **Seven of the ten witnesses would never run at a phase gate.** `yarn verify` → `turbo run test` → `vitest run --project unit`, and that project excludes `test/**/*.e2e.test.ts` (`vitest.config.ts:14-15`). | **The per-phase command gains `yarn workspace @jsvision/examples test:e2e`.** Recorded in AR-11, NFR-4, AC-7 and every phase's verify task. | ✅ Resolved |
