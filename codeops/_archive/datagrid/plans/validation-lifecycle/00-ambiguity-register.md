| AR-24 | Behavior / row gate (runtime) | **The row-leave gate keys on a new "touched-rows" registry, NOT `isRowDirty`.** 03-03/AR-5 keyed the gate on `isRowDirty()`, but that registry tracks only **in-flight** commits: it is set for the microtask around `await commitCell` and cleared the instant a commit resolves, and while a commit is pending the cell editor holds focus ‚Äî so the four row-leave gestures can never fire against a dirty row. The gate as written never traps (contradicting ST-12/13/15/16). Resolved (user-selected) with a `touched` set of row keys: a cell whose commit **succeeds** marks its row touched; a row-leave runs `validateRow` only for a touched row (untouched seed rows leave freely, preserving AR-5); a leave that passes clears the row from `touched` (so a re-entered valid row does not re-trap). `note`/`note(null)` (the Phase-2 keyless channel) surface/clear the message. The touched set lives beside `dirty`/`errors` in the container, marked via a `markRowTouched` EditHost seam. | `editing.ts` dirty lifecycle (add before `await commitCell`, delete after ‚Äî pending-only); `grid.spec` "pending async commit" test confirms dirty clears on resolve; the editor holds focus during a pending commit | ‚úÖ |
| AR-23 | Placement / guard (runtime) | **The `grid.ts` line guard re-bases to `< 1500`, not the plan's estimated `< 1350`.** AR-7 correctly predicted the guard crossing and mandated the re-base; its specific `< 1350` estimate proved low, and an interim `< 1450` (chosen after Phase 2 from a ~1407 projection) also proved low once Phase 4's lifecycle swap wiring landed.
