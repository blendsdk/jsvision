## How the gate was run

All 12 categories were swept against RD-08 + the real code (TV `teditor1/2.cpp`/`edits.cpp`/
`tmemo.cpp`/`tindictr.cpp`/`teditwnd.cpp`/`tfiledtr.cpp`/`textview.cpp`/`tvedit2/3.cpp` +
`app.h`/`views.h`/`dialogs.h`/`editors.h` palette chains; ui `desktop.ts`/`window.ts`/
`scroll-bar.ts`/`commands.ts`/`render-root.ts`/`view.ts`/`event-loop.ts`/`input.ts`;
files `fs/types.ts`; core `theme.ts`/`osc.ts`/`events.ts`). Most categories were already closed by
the RD/preflight (scope, buffer model, EOL policy, keymap precedence, security, deferrals, demos).
The open items cluster in: **Behavioral defaults** (undo depth + redo surface — PA-1; the
clipboard-editor default — PA-2), **Technical unknowns** (the drag-state seam — PA-3; the
segmentation module placement — PA-5; the `FileSystem` additions — PA-6; repaint granularity —
PA-9), **Scope** (Should-Haves in v1 — PA-4; the seam-default message boxes — PA-7), and
**Source pins** (the GATE-1 byte set — PA-8; gadget visibility — PA-10; the replace-prompt
geometry — PA-11; paste-selection semantics — PA-16). Zero items deferred.
