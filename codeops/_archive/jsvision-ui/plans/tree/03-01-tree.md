## TV decode (GATE 1)

### §1 · `graphChars` glyph table — `toutline.cpp:367`

```cpp
static const char* graphChars = "\x20\xB3\xC3\xC0\xC4\xC4+\xC4";
```

| Idx | CP437 | Glyph | Unicode (unambiguous-narrow) | Role |
|-----|-------|-------|------------------------------|------|
| 0 | 0x20 | (space) | U+0020 SPACE | Level filler (ancestor level with no continued sibling) |
| 1 | 0xB3 | `│` | U+2502 BOX DRAWINGS LIGHT VERTICAL | Level mark (ancestor level **with** a continued sibling below) |
| 2 | 0xC3 | `├` | U+251C BOX DRAWINGS LIGHT VERTICAL AND RIGHT | End graphic — node **has** a sibling below (non-last) |
| 3 | 0xC0 | `└` | U+2514 BOX DRAWINGS LIGHT UP AND RIGHT | End graphic — node is the **last** child |
| 4 | 0xC4 | `─` | U+2500 BOX DRAWINGS LIGHT HORIZONTAL | End filler (horizontal run before the marker) |
| 5 | 0xC4 | `─` | U+2500 | End filler (children variant; same glyph) |
| 6 | 0x2B | `+` | U+002B PLUS SIGN | Expand marker — **collapsed** node with children |
| 7 | 0xC4 | `─` | U+2500 | Expand marker — **expanded** node |

**No `[+]`/`[-]` brackets** — the marker is a bare `+` (collapsed) or `─` (expanded). All glyphs are
EAW-unambiguous width-1; the `+` is ASCII U+002B (not fullwidth U+FF0B). The project renders box/line
glyphs narrow (existing block-glyph convention).

### §2 · `getGraph` fixed widths — `toutline.cpp:364-370`

```cpp
char* TOutlineViewer::getGraph(int level, long lines, ushort flags) {
  static const int levelWidth = 3, endWidth = levelWidth;   // 3 and 3
  static const char* graphChars = "\x20\xB3\xC3\xC0\xC4\xC4+\xC4";
  return createGraph(level, lines, flags, levelWidth, endWidth, graphChars);
}
```
- **`levelWidth = 3`** — each ancestor level indents 3 columns.
- **`endWidth = 3`** — the end graphic (fork + filler + marker) is 3 columns.

### §3 · `createGraph` algorithm — `toutline.cpp:165-205`

Builds a row's prefix string. Flags (`outline.h:27-29`): `ovExpanded=0x01`, `ovChildren=0x02`,
`ovLast=0x04`.

**Phase 1 — ancestor level marks (`:181-186`):** for each of `level` ancestor levels (bit N of
`lines` = "level N has a continued sibling"): emit `chars[1]` (`│`) if the bit is set else `chars[0]`
(space), then `levelWidth-1` (= 2) spaces. → `level × 3` columns.

**Phase 2 — end graphic (`:188-201`), with `endWidth=3`:**
1. emit `last ? chars[3] (└) : chars[2] (├)`;
2. emit `1 × chars[4] (─)` filler;
3. emit `children ? chars[5] : chars[4]` (both `─`);
4. emit the marker: `expanded ? chars[7] (─) : chars[6] (+)`.

So a non-last node with collapsed children at level 1 renders `"   ├──+"` conceptually (3-space
ancestor indent + `├` + `─` + `─`/marker); an expanded one ends `─`. *(exact column composition is
re-diffed at GATE-2; the renderer builds the string via the same phases.)*

### §4 · `draw` / `drawTree` — `toutline.cpp:54-102`

Per visible node (`drawTree`, `:54-91`):
- **visibility** (`:61-64`): skip if `position < delta.y`; stop if `position ≥ delta.y + size.y`
  (virtual window).
- **row colour** (`:66-71`):
  ```cpp
  if ((position == foc) && (state & sfFocused))  color = getColor(0x0202);  // Focus
  else if (isSelected(position))                 color = getColor(0x0303);  // Select
  else                                           color = getColor(0x0401);  // Normal pair
  ```
- **clear row** (`:72`) then **draw graph** (`:73-79`) in `color`, at `x = strwidth(graph) - delta.x`.
- **node text, two-tone** (`:80-84`):
  ```cpp
  TColorAttr c = (flags & ovExpanded) ? color : (color >> 8);
  dBuf.moveStr(max(0,x), text, c, ...);
  ```
  **An expanded node's text uses the low byte of `color`; a collapsed node's text uses the high byte
  (`color >> 8`).** For the Normal pair `getColor(0x0401)`, low = slot 1 (Normal), high = slot 4
  (NotExpanded) — so a **collapsed** normal node's text is `outlineNotExpanded`. (Focus/Select rows
  are `0x0202`/`0x0303`, both bytes equal, so their text is single-tone.)

`draw()` (`:93-102`): `firstThat(drawTree)` walks nodes in display order, then fills the remainder of
the viewport with the Normal colour.

### §5 · Palette + `getColor` chain — `outline.cpp:15,351-355`, `outline.h:66-70`

```cpp
#define cpOutlineViewer "\x6\x7\x3\x8"                          // toutline.cpp:15
static TPalette p(cpOutlineViewer, sizeof(cpOutlineViewer));    // :353  (note: sizeof, not -1)
```
Palette layout (`outline.h:66-70`): **1 = Normal · 2 = Focus · 3 = Select · 4 = NotExpanded**.

**Storage rule** (`palette.cpp:23-28`): `data[0] = len; data[i+1] = string[i]` — 1-indexed. With
`sizeof` (5, incl. the null) the palette is `data = [5, 0x06, 0x07, 0x03, 0x08, 0x00]`; slots 1–4 =
`0x06/0x07/0x03/0x08`, slot 5 = `0x00` (never referenced — the missing `-1` is benign).

**Resolution** (`mapcolor.cpp:20-38`, pair split `tview.cpp:484-494`): `getColor(0xHHLL)` maps the
low byte through local index `LL`, the high byte through `HH`; `mapColor` recurses `view → owner →
app` to `cpAppColor` (`app.h:142-151`). The four slots' **local** parent indices are
`Normal=6 · Focus=7 · Select=3 · NotExpanded=8`.

> **Colour-byte pinning (PA-9):** the final attribute byte depends on the owner window palette. Per
> the 03-01 history-decode convention this plan fixes the **canonical host = gray `TDialog`** and
> pins the concrete `{fg,bg}` at the exec **GATE-1 BEFORE** task by walking the real chain, verified
> at **GATE-2**. The structure (4 slots, the three `getColor` calls, the two-tone split) is fully
> decoded above; only the last-hop bytes carry an exec-time confirmation. See
> [03-03](03-03-theme-packaging.md) for the best-effort decode + confidence. `cpAppColor` attribute
> convention: `0xHL`, high nibble `H` = background, low nibble `L` = foreground.

### §6 · Behaviour (TV originals; some overridden) — `toutline.cpp:419-541`, node methods `:640-695`

- **Keys** (`:484-540`): TV maps ← = up, → = down; `+` `adjust(node,True)` (expand), `-`
  `adjust(node,False)` (collapse), `*` `expandAll(node)`; Enter/Ctrl-Enter `selected(newFocus)`;
  PgUp/PgDn ±`size.y-1`; Home/End to `delta.y`±; Ctrl+PgUp/Dn to `0`/`limit.y-1`.
  **We override ←/→ to collapse/expand (PA-12)**; keep `+`/`-`/`*` + ↑↓ + paging faithful.
- **Mouse** (`:433-481`): a click with `mouse.x < strwidth(graph)` → `adjust(cur, !isExpanded(cur))`
  (toggle); a double-click → `selected(foc)`. **We map to PA-14** (no double-click): graph-zone click
  = toggle; text click = select+emit.
- **Node methods** (`:640-695`): `adjust` sets `expanded`; `isExpanded`/`hasChildren`/`getNumChildren`/
  `getChild`/`getNext` walk the child list; `cmOutlineItemSelected = 301` (`outline.h:32`).
- **Sizing** (`:563-594`): `update()` → `firstThat(countNode)` tallies `updateCount` (visible rows)
  + `updateMaxX` (widest row) → `setLimit(updateMaxX, updateCount)`. **We use `updateCount` for the
  vertical `ScrollBar` range** (H deferred, PA-5).

## jsvision design

### `TreeNode<T>` (in `tree.ts`, exported)
```ts
/** A plain, reactive-friendly tree node. Expand state is owned by the view, not the node (AR-141). */
export interface TreeNode<T> {
  readonly value: T;
  readonly children: TreeNode<T>[];
}
```

### `TreeOptions<T>` (in `tree.ts`, exported)
```ts
export interface TreeOptions<T> {
  roots: Signal<TreeNode<T>[]>;              // PA-2 forest
  getText: (value: T) => string;
  focused?: Signal<number>;                  // flattened-visible index (default signal(0))
  selected?: Signal<number>;                 // default signal(-1)
  onSelect?: (index: number, node: TreeNode<T>) => void;
  command?: string;                          // emitted on Enter / text-click (TV cmOutlineItemSelected)
  expandedByDefault?: boolean;               // PA-3 (default false = collapsed)
  guides?: boolean;                          // PA-6 (default true = │├└─ connectors)
}
```

### `Tree<T> extends Group` (in `tree.ts`)
- Layout `{ direction: 'row' }`; children = `rows: TreeRows<T>` (`size: fr`) + `bar: ScrollBar`
  (`size: fixed 1`, vertical), wired `this.rows.bar = this.bar` — mirrors `ListView` (`list-view.ts:75-77`).
- Owns the expand state: an object-reference `Set<TreeNode<T>>` + an `expandVersion = signal(0)`
  (PA-4). `expandedByDefault` seeds the Set (empty, or all nodes) at construction.
- Toggle helpers `expand(node)`/`collapse(node)`/`toggle(node)` mutate the Set + bump
  `expandVersion`. `expandAll()`/`collapseAll()` (PA-6) walk the forest + bump once.
- Exposes `rows` (the focus target) so a host can focus the Tree, like `ListView.rows`.

### `TreeRows<T> extends View` (in `tree-rows.ts`, the ≈450-line renderer)
- `focusable = true`; holds `getText`, `focused`/`selected`, `command`/`onSelect`, `guides`, the
  Tree's `flatten()` accessor + expand predicate/toggle callbacks, `topItem`, `bar?`.
- **`draw(ctx)`**: read the flattened rows; `bar?.setRange(0, max(0, n-1), max(1, rows-1))`;
  `topItem = keepVisible(focused, topItem, rows, n)`; for each visible row build the graph prefix via
  `graph.ts` `createGraph(level, lines, flags, guides)` and draw prefix + `getText(node.value)`;
  resolve the row role **focused > selected > normal**, and the text colour **two-tone** (collapsed →
  `outlineNotExpanded`) per §4.
- **`onEvent`**: wheel ±3; mouse-down → `row = mouse.y + topItem`, focus it, then
  `mouse.x < graphWidth(row)` ? toggle-expand : select+emit (PA-14); keys ↑↓/Pg/Home/End/Ctrl+Pg
  (RD-11-style), `+`/`-`/`*`, ←/→ (PA-12), Enter (select+emit).
- After any expand/collapse, `focused` is re-clamped via `clampIndex(focused, newCount)` (PA-15).

### Flatten (in `graph.ts`, see [03-02](03-02-graph-and-model.md))
`flattenVisible(roots, isExpanded) → FlatRow<T>[]` where
`FlatRow = { node, level, lines, flags }` (the `createGraph` inputs). Depth-guarded iteration
(AC-13). The renderer and the ScrollBar range both read this list.
