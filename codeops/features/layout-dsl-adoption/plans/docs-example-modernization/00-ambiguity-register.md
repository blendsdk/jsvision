| AR-10 | Design | How an allowlist entry is keyed — the sweep shifts line numbers | (a) `file::symbolName`; (b) `file#ordinal`; (c) `file:line`, regenerated | **(a) `file::symbol`, qualified and ordinal-suffixed where the symbol alone is not unique** | The `@example` is attached to a declaration, so the harness already knows the symbol. Stable across line shifts and reorderings, and it reads meaningfully in review. (b) silently re-targets when an example is inserted above it — the one failure mode a ratchet must not have. **Preflight amendment — the bare symbol is not unique.** Three real collisions exist: (i) two `@example` tags on the **same** declaration (`packages/ui/src/controls/input.ts:47` and `:58`, both on `class Input` — a file this plan edits); (ii) member-level blocks keyed by a bare method name, which is not file-unique by construction (39 such blocks: 20 `MethodDeclaration`, 8 `MethodSignature`, 11 `PropertySignature`); (iii) a genuine anonymous block (`packages/files/src/fs/node-fs.ts:9`, an `@example` on the file's leading comment, which binds to the first `ImportDeclaration`). The key is therefore `file::Symbol`, with a `Class.member` qualifier for members and a `#N` suffix (1-based, in source order) **only when a key would otherwise repeat**. `(anonymous)` follows the same rule → `(anonymous)#1`. The `#N` suffix is scoped *within one symbol*, so AR-10's objection to (b) — file-wide ordinals silently re-targeting — does not apply |

| AR-13 | Design *(measured, superseded by AR-16)* | Examples that use a **relative** import (`'./button-row.js'`) cannot resolve from a scratch directory | — | **Each block must be compiled *as if* it sat beside its own source file** | Measured: compiling blocks from a shared temp dir produced ~22 phantom `TS2307 Cannot find module './x.js'` failures. (b) would put all 56 permanently on the allowlist and inflate the drain issue with non-defects; it would also invalidate the AC-6 oracle table, whose `spinner.ts::Spinner` row records `TS2304` where base-as-written reports `TS6133` first. **This was a decision the plan was making implicitly and unknowingly** — the feasibility probe ran with these checks off, which is why its numbers reproduce (a) exactly |
| AR-15 | Scope | Which `src/` trees the guard walks — FR-1's `packages/*/src` glob, 03-01's seven-package enumeration, and docs-site's own four-package boundary all disagreed | (a) the six shipped packages, enumerated; (b) the glob as written; (c) docs-site's four documented packages | **(a) `core, ui, web, files, datagrid, forms` — an explicit enumeration, never a glob** | The glob also matches `packages/docs-site/src` (12 blocks) and `packages/spike-data-studio/src` (2), the latter a package CLAUDE.md marks "delete after the Data Studio decision" — seeding a permanent ratchet from an inert package is wrong. `theme-designer` is dropped too: it is an **app**, not shipped source (CLAUDE.md scopes the JSDoc directive to shipped packages), and it has **no `build` script**, so it has no `dist` for its blocks to resolve against. | AR-16 | Design *(preflight)* | Where the extracted blocks physically live while they compile. AR-13 wrote real `.ts` files into `packages/<pkg>/src/`, beside real sources | (a) an in-memory `ts.CompilerHost`; (b) on-disk beside the source, hardened; (c) a sibling directory plus a `paths` shim | **(a) in-memory `CompilerHost`** | The on-disk write was the guard's single largest hazard and it failed three ways: it races turbo's `typecheck`/`build` (which have **no ordering** against `test`), a leak is **silent** (verified — `tsc`'s wildcard `include` does not match dot-prefixed filenames, and `.gitignore` has no entry), and a `finally` cannot cover SIGINT/SIGKILL. It also removes the intra-vitest race between the spec run, the impl run and AC-9's forced-failure run. (b) is a complete fallback but every one of its controls guards a path nobody exercises; (c) re-opens the ~22 phantom `TS2307`s AR-13 exists to close |
